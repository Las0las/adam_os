# LIS-002 — Canvas vs. Shelf Layout Shell

IDE-grade layout engine for the LAWRENCE platform. Strictly segregates **active
workspaces (the Canvas)** from **metadata / navigation control surfaces (the
Shelf)**, so the screen shows work — not platform complexity — until you ask for it.

## Components

- `WorkspaceShell` — `src/components/runtime-console/lis-shell.tsx`
- `useLayoutMemory` — same module (persistence hook)
- Engine styles — `src/components/runtime-console/lis-shell.css`
- Demo — `/workspace-shell` (`WorkspaceShellDemo.tsx`, surfaced under Studios)

## Contract

```tsx
interface WorkspaceShellProps {
  canvas: React.ReactNode;          // owns the primary viewport
  leftRail?: React.ReactNode;       // Shelf — navigation
  rightInspector?: React.ReactNode; // Shelf — metadata
  bottomTerminal?: React.ReactNode; // Shelf — terminal
  layoutState: 'minimized' | 'expanded-overlay' | 'expanded-flex';
  initialState?: ViewportLayoutState;          // hydrate from memory
  onLayoutChange?: (s: ViewportLayoutState) => void;
}
```

## System rules

1. **Canvas owns the viewport.** CSS Grid: `var(--lis-lc) minmax(0,1fr) var(--lis-rc)`.
   The center track is the only flexible one; it centers + fills dynamically.
2. **Shelf baseline = exactly 40px**, icon-only. Closed rails render only their
   strip/bar; the panel is `display:none`, so the footprint is zero beyond 40px.
3. **Two expansion modes** (per `mode`, toggled live via the
   `lis-shell:set-mode` window event, or seeded by `layoutState`):
   - `overlay` — rail track stays 40px; the panel is `position:absolute` and
     floats **over** the canvas (+ scrim, Esc / click-to-close). The canvas
     never shifts or squishes.
   - `flex` — the grid track grows to `40 + panelSize`; the canvas reflows with
     **hard structural containment** (`overflow:hidden`, `minmax(0,1fr)`).
4. **State persistence.** Every toggle / drag / resize / mode / viewport-resize
   emits a serializable `ViewportLayoutState` to `onLayoutChange`.
   `useLayoutMemory(key, fallbackPosture)` hydrates `initialState` from
   `localStorage` (corrupt payloads fall back) and writes each payload back.

## Notes

- Track lengths animate via registered `@property --lis-lc/-rc/-br` custom
  properties; easing is disabled (`data-dragging`) during an active drag so
  resize tracks the pointer 1:1, and under `prefers-reduced-motion`.
- Drag handles are `role="separator"` with orientation + labels; strip buttons
  carry `aria-expanded` and one shared focus ring.
- Self-contained: the shell re-declares the LIS motion tokens, so it works
  outside the runtime console too.
