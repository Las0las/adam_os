# LIS-003 — Three-Layer Navigation Mesh

Status: **Prototype shipped** · Route `/nav-mesh` · Studios nav

A unified, IDE-grade navigation prototype that wires three navigation layers to **one
source of truth** (`nav-store.ts`, Zustand). Any layer mutates the store; every layer
re-renders from it. No layer owns state.

## The three layers

| Layer | Surface | Mutates |
|-------|---------|---------|
| 1 · Workspace Rail | 60px left rail of enterprise **objects** | `pivotTo(object, id, label)` |
| 2 · Surface Tabs | header tab bar of canonical **lenses** | `setSurface(surface)` / `togglePinSurface` |
| 3 · Omnibar | ⌘K command palette | `pivotTo` / `setSurface` (atomic) |

## Canonical surfaces (lenses)

`overview · grid · document · canvas · composer · ai_studio · relationships · evidence · history`

## Architectural guarantees

- **Atomic context pivot** — `pivotTo` changes object + instance + surface in a single
  `set()` tick (`pivotTick` increments once), so an omnibar command like
  `@Sarah Chen /canvas` never produces an intermediate render.
- **Read-your-context** — `surfaceHistory[id]` restores the last lens you used for an
  instance when you pivot back to it.
- **Non-destructive surface caching** — `SurfaceProjectionContainer` keeps every surface
  mounted and toggles `display:none`. Scroll position, input focus, textarea undo history
  and streaming buffers survive a lens switch (proven by the live ticker on the Grid
  surface and the persistent Document textarea).
- **Keyboard** — ⌘K toggles the omnibar; `Alt+1..6` cycle the first six surfaces; Esc
  closes; tabs and pins are keyboard-operable.

## Omnibar grammar

- **Model A — slash pivot** (lens only): `/grid`, `/document`, `/ai`, `/canvas`, …
- **Model B — natural compilation** (entity + optional lens): `@Sarah Chen /canvas`,
  `revenue`, `aberdeen`.

## Pattern 2 — In-line projections & ghost text (`inline-projector.tsx`)

A zero-layout-thrash inline diff engine layered into the Grid and Document surfaces.
State is exposed as `data-state` so the stylesheet (not JS) owns the look, and it inherits
the LIS-001 state tokens:

| status | render | data-state |
|--------|--------|-----------|
| `idle` | base value, no decoration | `idle` |
| `streaming` | ghost text (dim + italic) with a blinking caret; an invisible base reserves the box so nothing reflows | `runtime-executing` |
| `optimistic` | speculative deletion (strike-through) + insertion (dashed underline) inline | `speculative` / `speculative-deletion` / `speculative-insertion` |
| `rejected` | reverts to base with a brief flash | `rejected` |

## Adaptations from the pasted blueprint

- Inline SVG icons (the platform has no icon library) instead of emoji glyphs.
- Scoped CSS tokens instead of raw Tailwind hex, consistent with LIS-001/002.
- Middle-click "detach frame" `alert()` replaced by a real, accessible pin affordance.
